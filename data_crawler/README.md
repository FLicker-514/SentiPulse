# SentiPulse 数据采集说明文档

## 一、数据来源

### 1. 股票数据 — A股日线行情

| 项目 | 说明 |
|------|------|
| **数据来源** | AKShare (基于东方财富数据接口) |
| **备用来源** | Baostock (证券宝) |
| **数据类型** | A股日线 K 线数据 (OHLCV) |
| **复权方式** | 前复权(qfq) / 后复权(hfq) / 不复权(none) |
| **获取方式** | `akshare.stock_zh_a_hist()` |
| **数据字段** | 日期、开盘价、最高价、最低价、收盘价、成交量、成交额、振幅、涨跌幅、涨跌额、换手率 |

> AKShare 是开源 Python 金融数据接口库 (GitHub 16800+ Stars)，底层调用东方财富公开数据接口。无需注册或 API Key。Baostock 作为备用方案，在 AKShare 不可用时自动切换。

### 2. 新闻数据 — 东方财富 (Eastmoney) 新闻搜索

| 项目 | 说明 |
|------|------|
| **数据来源** | 东方财富 (Eastmoney) 新闻搜索 API |
| **API 端点** | `search-api-web.eastmoney.com/search/jsonp` |
| **数据聚合** | 聚合自 证券时报、中国证券报、上海证券报、证券日报 等权威财经媒体 |
| **搜索方式** | 按股票代码 + 公司名称分别搜索 |
| **数据类型** | 公司相关新闻报道 |
| **获取方式** | JSONP API, 无需认证, 返回结构化 JSON |
| **数据字段** | 标题、URL、发布时间、来源、摘要、文章类型 |

> 东方财富搜索 API 聚合了包括证券时报在内的多家权威财经媒体新闻。该 API 为公开接口，无需注册或 API Key。每次搜索最多返回约 500-700 条结果，通过股票代码和公司名称双关键词搜索最大化数据量。

### 3. 公司公告 — 巨潮资讯 (Cninfo)

| 项目 | 说明 |
|------|------|
| **数据来源** | 巨潮资讯网 (cninfo.com.cn) — 中国证监会指定信息披露平台 |
| **API 端点** | `cninfo.com.cn/new/fulltextSearch/full` |
| **搜索方式** | 按公司名称 + 股票代码分别搜索 |
| **数据类型** | 上市公司法定披露公告 (董事会决议、定期报告、回购进展、重大合同等) |
| **获取方式** | HTTP GET, 无需认证, 返回 JSON, 支持日期范围过滤 |
| **覆盖范围** | 可追溯多年, 本数据集覆盖 2025-01-01 至今 |

> 巨潮资讯是 A 股上市公司法定信息披露平台。所有重大事项均须在此公告。数据具有权威性、完整性和法律效力。

## 四、数据存储

### 数据库：SQLite

- **文件路径**: `data/sentipulse.db`
- **存储引擎**: SQLite 3 (WAL 模式)
- **字符编码**: UTF-8

### 表结构

#### `stock_daily` — 股票日线数据

| 字段 | 类型 | 说明 |
|------|------|------|
| stock_code | TEXT | 股票代码 (如 600519) |
| stock_name | TEXT | 企业名称 (如 贵州茅台) |
| trade_date | TEXT | 交易日期 (YYYY-MM-DD) |
| open | REAL | 开盘价 |
| high | REAL | 最高价 |
| low | REAL | 最低价 |
| close | REAL | 收盘价 |
| volume | REAL | 成交量 |
| amount | REAL | 成交额 |
| amplitude | REAL | 振幅(%) |
| pct_change | REAL | 涨跌幅(%) |
| change | REAL | 涨跌额 |
| turnover | REAL | 换手率(%) |
| adjust | TEXT | 复权类型 (qfq/hfq/none) |
| fetched_at | TIMESTAMP | 数据采集时间 |

> 唯一约束: `(stock_code, trade_date, adjust)`

#### `news_articles` — 新闻数据

| 字段 | 类型 | 说明 |
|------|------|------|
| company_name | TEXT | 关联企业名称 |
| search_keyword | TEXT | 搜索关键词 |
| title | TEXT | 新闻标题 |
| url | TEXT | 新闻原文链接 |
| publish_time | TEXT | 发布时间 |
| source | TEXT | 新闻来源 |
| excerpt | TEXT | 摘要/节选 |
| tags | TEXT | 标签 (逗号分隔) |
| article_type | TEXT | 文章类型 (kx=快讯) |
| full_content | TEXT | 快讯全文内容 |
| fetched_at | TIMESTAMP | 数据采集时间 |

> 唯一约束: `(url)`

#### `crawl_progress` — 采集进度

| 字段 | 类型 | 说明 |
|------|------|------|
| task_type | TEXT | 任务类型 (stock/news) |
| task_key | TEXT | 任务标识 (股票代码/企业名) |
| status | TEXT | 状态 (pending/running/completed/failed) |
| total_items | INTEGER | 已采集条目数 |
| last_error | TEXT | 最后一次错误信息 |

## 五、反爬虫与速率控制

| 措施 | 说明 |
|------|------|
| **请求间隔** | 股票: 0.8-1.5s / 新闻: 2.0-4.0s |
| **User-Agent 轮换** | 5 个真实浏览器 UA 随机轮换 |
| **Referer 伪装** | 每次请求携带合适的 Referer |
| **自动重试** | 最多 3 次, 指数退避 (5s/10s/15s) |
| **断点续采** | 通过 `crawl_progress` 表记录进度, 中断后自动跳过已完成任务 |
| **URL 去重** | 新闻按 URL 唯一约束自动去重 |
| **日志记录** | 所有操作记录到 `logs/crawler.log`, 支持轮转 |

## 六、使用方法

### 环境准备

```bash
cd data_crawler
uv sync                    # 安装依赖
```

### 运行采集

```bash
# 采集全部数据 (股票 + 新闻)
uv run python main.py

# 仅采集股票数据
uv run python main.py --stock

# 仅采集新闻数据
uv run python main.py --news

# 查看当前数据统计
uv run python main.py --status
```

### 静默后台运行 (Windows)

```powershell
# 后台运行, 日志输出到文件
Start-Process -NoNewWindow -FilePath "uv" -ArgumentList "run python main.py" -RedirectStandardOutput "logs/stdout.log" -RedirectStandardError "logs/stderr.log"
```

### 输出文件

| 文件/目录 | 说明 |
|-----------|------|
| `data/sentipulse.db` | SQLite 数据库 (股票+新闻) |
| `logs/crawler.log` | 详细运行日志 |
| `logs/stdout.log` | 标准输出 |
| `logs/stderr.log` | 错误输出 |

## 八、注意事项

1. **网络环境**: 需要能正常访问 stcn.com 和 Eastmoney 数据接口
2. **运行时间**: 全量采集预计需要 20-60 分钟（取决于网络和间隔设置）
3. **数据合规**: 采集的均为公开数据，仅供研究学习使用
4. **盘中采集**: 股票数据建议在收盘后(15:00后)采集以确保当日数据完整
5. **增量更新**: 重新运行会自动跳过已有数据(URL/日期去重)，仅采集新增部分
